# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability in `django-sabia-auth`, please report it responsibly.

### How to Report

**Preferred method:** Use [GitHub Private Security Disclosure](https://github.com/kelsoncm/django-sabia-auth/security/advisories/new) to report the vulnerability privately.

**Alternative:** Send an email describing the vulnerability. Include "SECURITY" in the subject line.

### What to Include

Please include the following in your report:

- A clear description of the vulnerability
- Steps to reproduce the issue
- The potential impact (e.g., data exposure, authentication bypass)
- Any suggested mitigations or fixes
- Your contact information for follow-up

### Response Timeline

- **Acknowledgement:** Within 48 hours of receiving your report
- **Initial assessment:** Within 5 business days
- **Resolution target:** Within 30 days for critical issues, 90 days for others

### Disclosure Policy

- We will work with you to understand and validate the issue
- We will prepare a fix and coordinate a release timeline with you
- We will publicly disclose the vulnerability after a fix is available
- We credit reporters in the changelog unless you prefer to remain anonymous

Thank you for helping keep `django-sabia-auth` and its users safe.
